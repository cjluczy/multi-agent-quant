# 从概念到可运行控制台：Multi-Agent Quant 项目实践记录

## 一、为什么要做这个项目

很多“AI 量化”项目停留在两个层面：

- 只有策略概念，没有真正跑起来的系统闭环
- 只有回测结果，没有运行控制台、持仓跟踪和风险可视化

这个项目想做的是另一件事：把多智能体量化系统真正做成一个可以运行、可以看、可以手动调参、可以跟踪历史的控制台产品。

目标不是一步到位做成真正实盘系统，而是先把下面这条链路打通：

`行情 -> 智能体 -> 风控 -> 执行 -> 持仓 -> 盈亏 -> 历史`

## 二、项目里到底有哪些角色

系统把策略不再理解成一个单点模型，而是拆成多个有职责分工的智能体：

- 新闻事件型智能体，负责对外部事件和市场情绪变化更敏感
- 反应式交易智能体，负责更短周期的价格反应
- 波段型智能体，负责更平滑的趋势判断
- 对冲型智能体，负责期货或组合对冲
- 均值回归型智能体，负责偏反转机会
- 突破型智能体，负责趋势突破场景
- 基本面型智能体，负责更稳健的选股倾向

这些智能体不是简单地堆在一起，而是通过组合大脑、风控引擎和执行层被统一约束。

## 三、这次补上的关键能力是什么

这轮改造里，真正重要的不是“页面更漂亮”，而是把真实行情接到了可用状态。

### 已打通的真实行情入口

- `EasyQuotation`：本地可直接启用，适合作为默认实时行情入口
- `Tushare`：支持切换到实时行情，但需要用户自备 Token
- `CSV 回放`：可把外部真实市场数据导入系统做准实盘流程验证

### 为什么默认选 EasyQuotation

原因很现实：

- 本地依赖就绪
- 不需要额外申请 Token
- 可以更快验证真实行情链路是否打通

在本地验证中，系统已经能通过 `EasyQuotation` 拉到真实行情，并用它驱动整个运行流程，而不是停留在模拟行情。

## 四、控制台做了什么改动

为了让它更像一个“可操作的系统”而不是“单页 Demo”，控制台重点补了这些能力：

- 可以在页面中手动修改实时行情源
- 可以直接配置 `Tushare Token`
- 可以配置 CSV 路径和字段映射
- 可以修改系统模式、轮询频率、初始资金和风险预算
- 可以增删智能体、调整角色和资金占比
- 可以直接发起一次新的运行
- 可以在页面中看到当前实际生效配置，而不是只看静态默认值

这意味着控制台现在不只是展示面板，而已经是一个轻量级的策略运行驾驶舱。

## 五、真实有效，和可实盘，差在哪里

这是最容易被误解的一点。

接入真实行情，并不等于系统已经可以直接拿去实盘。

目前它已经很接近“真实有效”的研究和纸面交易环境，因为这些核心环节已经具备：

- 真实行情输入
- 多智能体决策协同
- 风控限制
- 纸面执行
- 持仓、盈亏与历史追踪

但距离真正“可实盘”，还差几个关键层：

- 真实券商或交易柜台接入
- 更完整的交易时段控制与异常状态处理
- 更贴近真实市场的撮合与委托生命周期
- 人工确认、风控告警、审计日志和熔断机制
- 实盘级容灾、监控和回滚能力

换句话说：

它现在已经不只是 PPT 系统，但也还不是可以不看着就真金白银自动交易的系统。

## 六、项目现在更适合做什么

现阶段，它最适合用来做三件事：

- 做实时纸面交易和策略观察
- 做多智能体协同策略实验
- 做接券商前的产品化中间层验证

如果你要继续往“可实盘”逼近，最优路径通常是：

1. 保持真实行情接入
2. 持续用纸面执行验证策略与风控
3. 接入真实基本面、新闻和交易日历
4. 再接仿真柜台或 broker API
5. 最后才考虑有限实盘

## 九、结语

这个项目最有价值的地方，不在于“是不是已经完全实盘”，而在于它已经跨过了很多 AI 量化项目最难跨的一步：

它不是只有想法，而是已经有了真实行情驱动下的可运行系统形态。

接下来继续补真实数据、执行接口和生产级风控，它就能从“研究系统”逐步变成“接近实盘的交易系统”。
